# Content model

## Purpose

This document describes the technical model for editable content in the ReduceCO2Now application.

## Possible content areas

The application may need to represent:

- pages
- news items
- events
- campaigns or initiatives
- resources
- uploaded media
- contact or participation enquiries

## Questions for later technical decision

- Which content types are stored as editable records?
- Which content types are static or file-based?
- Which content types need draft, review or publication states?
- Which content types need translation or localization support?
- Which content types involve personal data?
