## Date

## Status

## Context

The developer has an existing web application project whose structure, development habits and documentation patterns may be useful for ReduceCO2Now.

ReduceCO2Now is expected to need similar broad capabilities, including public pages, logged-in administration, editable content, roles and workflows.

## Decision

Use the existing project as a reference and pattern library.

Do not copy it blindly. Reuse patterns where they fit the ReduceCO2Now domain and discard anything that would add unnecessary complexity.

## Useful reference areas

- public page structure
- administrative dashboard concepts
- role-based access
- editable content workflows
- multilingual readiness
- technical documentation habits
- deployment and maintenance habits

## Risks

- copying unnecessary complexity too early
- letting the old project's domain shape the new application too strongly
- making technical choices before organisational needs are clear
