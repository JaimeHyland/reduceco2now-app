# Admin structure

## Purpose

This document describes the technical structure of the logged-in administration area.

## Expected admin capabilities

The administration area is expected to support some or all of the following:

- user management
- role and permission management
- editable content
- event management
- campaign or initiative management
- resource management
- workflow support for non-technical users

## Notes

This document should record implementation structure, not Scrum planning or user-story detail.

Detailed product requirements should remain in the external product and backlog documents.
